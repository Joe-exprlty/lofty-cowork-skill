Note on data handling. When you ask Claude to look up, update, or summarize leads, the lead content Claude reads (names, contact info, notes, activity) is processed through Anthropic's models the same way any Claude chat is processed. Setup

Step by step

Six short steps. Claude runs in Easy Mode by default and handles everything technical for you. If you'd rather move fast and skip the explanations, tell Claude "I'm technical" or "skip ahead" and it'll switch to Power User Mode. Works the same on Mac, Windows, and Linux. Claude detects your operating system and adjusts.

1

Download the skill file

Save it somewhere you can find. About 42 KB.

Download lofty-cowork-helper.skill
2

Install the skill in Claude Desktop

The easy way: double-click the lofty-cowork-helper.skill file you just downloaded. Claude Desktop should pop open with a dialog that says "Add lofty-cowork-helper to your library?" Click Add to library and you're done.

If double-click doesn't open Claude: open Claude Desktop. In the left sidebar, click Cowork, then click the Customize button. You'll see a Skills section there. Drag the .skill file into it (or use the upload option if your version has one). This fallback sometimes matters on Linux or right after a fresh Claude install, before the file association registers.
3

Confirm it installed

Open a new Cowork conversation and ask:

Do you have a Lofty skill installed?

Claude should mention lofty-cowork-helper.

4

Let Claude create your project folder

Claude creates a folder called lofty-tools inside your home Code folder. No command needed from you.

The only thing you do: in Claude Desktop, point Cowork at that folder when Claude asks you to. If Cowork is already open, Claude will tell you exactly where to click.

What's a "Cowork folder"? It's just the folder Claude is allowed to read and write files in for this conversation. Think of it as Claude's desk for this project.
Where your files end up (good to know for later):
On Mac and Linux: ~/Code/lofty-tools/ (which expands to /Users/yourname/Code/lofty-tools/ on Mac).
On Windows: C:\Users\yourname\Code\lofty-tools\.
Keep this path in mind. When we add showing reminders, post-showing texts, or form-to-CRM automations later (Phase 2), all of that builds inside this same folder. Your Lofty key, your customizations, and any new tools Claude adds will live here.
5

Ask Claude to run setup

In your Cowork session, say:

Set up Lofty for the first time.

Claude takes it from there. It will ask you a few short questions one at a time (your name, brokerage, phone, email, timezone), walk you through getting your Lofty API key, save it for you, and run a connection test. About 10 minutes total.

Heads up about your Lofty key: Treat it like a password. Don't paste it into chat. Claude saves it to a small settings file on your own computer (in the lofty-tools folder from Step 4). It is not uploaded anywhere, and it is not stored in the GitHub repo. The GitHub repo only holds the skill code that's the same for everyone. Your key stays local to your machine, period.
6

Try it with a real lead

Once setup finishes, Claude offers a guided demo using your actual data. Just tell Claude the name of one of your leads. Claude will pull them up, show you what they've been doing on your site, and walk you through your first real action.

If you want to test on your own, try these prompts:

Find my most recent Lofty leads
Show me Jane Smith's recent activity

Reference

Top 5 Lofty quirks

The skill handles all of these for you. Knowing them helps when an error appears.

  1. Auth header is token, not Bearer. Mixing them up returns error 200058.
  2. /v1.0/leads ignores keyword and sort. The skill uses a leads index workaround.
  3. Activities must use v1.0. The v2.0 endpoint returns empty results.
  4. Notes go to POST /v1.0/notes with leadId in body. The intuitive path returns 404.
  5. Rate limit is 10 requests per minute. The starter client enforces 6.5 second spacing.
Safety

Habits we recommend

Built into how the skill operates. Knowing them helps you understand what Claude is doing.

  • Confirm before email or text. Always. Claude drafts and asks before sending.
  • Confirm before deleting. The Lofty UI does not have an undo.
  • Never paste your API key into chat. Treat it like a password.
  • Verify the lead ID before logging notes. A name match could be the wrong person.
  • Times use ISO 8601 with offset. Pacific is -07:00 in DST, -08:00 outside.
